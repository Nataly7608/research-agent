2. ГДЕ ЛЕЖАТ НАСТРОЙКИ
--------------------------------------------------------------------------------

  Настройки проекта находятся в ДВУХ уровнях:

  УРОВЕНЬ 1 — opencode.json (корень проекта)
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  Файл:       research-agent/opencode.json
  Роль:       Центральный конфигурационный файл проекта OpenCode.
              Определяет агента, MCP-серверы, разрешения и навыки.
  Формат:     JSON

  ЧТО СОДЕРЖИТ:
  ┌─────────────────────────────────────────────────────────────────────────┐
  │  Поле                   │  Значение                │  Описание          │
  ├─────────────────────────┼──────────────────────────┼────────────────────┤
  │  $schema                │  URL на opencode.ai       │  Схема валидации   │
  │  default_agent          │  "researcher"             │  Агент по умолч.   │
  │  instructions           │  ["AGENTS.md"]            │  Загружаемые       │
  │                         │                          │  инструкции        │
  │  agent.researcher       │  { description, mode,    │  Конфигурация      │
  │                         │    permission }           │  агента-исслед.    │
  │  agent.researcher.mode  │  "primary"                │  Основной агент    │
  │  agent.researcher.      │  { edit: allow,           │  Разрешения        │
  │    permission           │    bash: ask,             │  (права доступа)   │
  │                         │    read: allow,           │                    │
  │                         │    websearch: allow,      │                    │
  │                         │    webfetch: allow }      │                    │
  │  mcp.playwright         │  { type: "local",         │  MCP-сервер        │
  │                         │    command: [...],        │  Playwright        │
  │                         │    enabled: true }        │  (веб-скрапинг)    │
  │  skills.paths           │  [".opencode/skills"]     │  Путь к навыкам    │
  └─────────────────────────────────────────────────────────────────────────┘

  УРОВЕНЬ 2 — .opencode/agents/researcher.md (frontmatter + тело)
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  Файл:       research-agent/.opencode/agents/researcher.md
  Роль:       Описание и инструкции для агента-исследователя.
              Дублирует и детализирует permission из opencode.json.
  Формат:     Markdown с YAML-frontmatter

  ЧТО СОДЕРЖИТ:
  ┌─────────────────────────────────────────────────────────────────────────┐
  │  Frontmatter (YAML):                                                    │
  │    description > "Проводит исследования, анализирует..."                │
  │    mode:       primary                                                  │
  │    permission:                                                          │
  │      edit:      allow                                                   │
  │      bash:      ask                                                     │
  │      read:      allow                                                   │
  │      websearch:  allow                                                  │
  │      webfetch:   allow                                                  │
  │                                                                         │
  │  Основное тело:                                                         │
  │    1. Роль агента                                                       │
  │    2. Правила безопасности                                              │
  │    3. Правило нейминга файлов                                           │
  │    4. Правила работы                                                    │
  └─────────────────────────────────────────────────────────────────────────┘


3. РАСШИФРОВКА НАСТРОЕК permissions
--------------------------------------------------------------------------------

  Разрешения (permissions) контролируют, какие действия агент может
  выполнять без подтверждения пользователя:

  ┌──────────────────────┬──────────┬──────────────────────────────────────┐
  │  Инструмент          │  Режим   │  Что означает                        │
  ├──────────────────────┼──────────┼──────────────────────────────────────┤
  │  read  (чтение)      │  allow   │  Можно читать любые файлы проекта    │
  │  edit  (запись)      │  allow   │  Можно создавать и редактировать     │
  │                      │          │  файлы в рабочей директории          │
  │  websearch (поиск)   │  allow   │  Можно искать в интернете            │
  │  webfetch (запросы)  │  allow   │  Можно загружать контент с URL       │
  │  bash (shell)        │  ask     │  Нужно спрашивать пользователя       │
  │                      │          │  перед запуском команд               │
  └──────────────────────┴──────────┴──────────────────────────────────────┘


4. MCP-ПОДКЛЮЧЕНИЕ
--------------------------------------------------------------------------------

  MCP-сервер:       Playwright
  Тип:              local (локальный запуск)
  Команда запуска:  npx -y @playwright/mcp
  Статус:           enabled (включён)
  Назначение:       Веб-скрапинг, браузерная автоматизация

  Доступные инструменты (22 шт):
    playwright_browser_navigate, _snapshot, _click, _type,
    _fill_form, _select_option, _hover, _take_screenshot,
    _evaluate, _network_requests, _network_request,
    _console_messages, _press_key, _drag, _drop,
    _file_upload, _handle_dialog, _resize, _tabs,
    _wait_for, _navigate_back, _close, _run_code_unsafe
